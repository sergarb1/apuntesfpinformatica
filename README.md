## 🎯 Qué es

**Apuntes FP Informática** es un **directorio curado de recursos educativos** para la familia profesional de Informática y Comunicaciones. Recopila apuntes, cursos, cheatsheets, juegos, retos, libros, jueces online y máquinas virtuales — todo en un solo sitio, accesible desde cualquier dispositivo.

| Colectivo | Beneficio |
|-----------|-----------|
| 👨‍🏫 **Profesorado** | Material listo para usar en clase, inspiración para tus unidades didácticas y referencias para el alumnado |
| 👩‍🎓 **Alumnado** | Enlaces a apuntes, tutoriales interactivos, jueces de programación, cursos gratuitos y más |
| 👥 **Comunidad educativa** | Un punto de encuentro para compartir y descubrir recursos de calidad |

Todo en una **SPA rápida, sin publicidad, sin registro y gratuita**.

## ✨ Características

- **🔍 Búsqueda en tiempo real** — Filtra por nombre y descripción al instante
- **📂 Filtros por módulo y tipo** — Encuentra lo que necesitas en segundos
- **♾️ Scroll infinito** — Navega sin paginación
- **📱 Responsive** — Funciona en móvil, tablet y escritorio
- **⚡ Sin servidor** — HTML + JSON estático, servible desde cualquier sitio
- **🔗 Dos dominios** — Disponible tanto en la [web oficial](https://apuntesfpinformatica.es) como en [GitHub Pages](https://sergarb1.github.io/apuntesfpinformatica/)
- **🌐 Internacional** — Recursos en castellano, catalán e inglés

## ⚡ Tecnología

```
📄 index.html      →   Vue 3 (CDN) + Tailwind CSS + Lucide Icons
📦 recursos.json   →   Base de datos de recursos (~290 KB)
🖼️ logo.png        →   Logo del sitio
```

| Herramienta | Propósito |
|-------------|-----------|
| [Vue 3.4](https://vuejs.org/) | SPA reactiva reactiva sin compilación, desde CDN |
| [Tailwind 3.4](https://tailwindcss.com/) | Estilos utilitarios inline desde CDN |
| [Lucide](https://lucide.dev/) | Iconos SVG ligeros y modernos |

> **Sin build step, sin package manager, sin servidor.**
> Abres `index.html` en el navegador y funciona.

## 🧑‍💻 Desarrollo

### Añadir un recurso

Edita `recursos.json` y añade una entrada con esta estructura:

```json
{
  "Título": "Nombre del recurso",
  "URL": "https://ejemplo.com",
  "Descripción": "Descripción clara y útil del recurso.",
  "Autor": "Nombre del autor",
  "Github": "https://github.com/autor/repo",
  "Twitter": "",
  "Licencia": "CC BY-SA 4.0",
  "Etiquetas": ["PRG-APUNTES"],
  "FechaPublicacion": "2026-07-08",
  "Modulos": ["PRG"]
}
```

### Convenciones de etiquetas

Formato: `{MODULO}-{TIPO}`

| Tipo | Etiqueta |
|------|----------|
| Apuntes propios | `PRG-APUNTES` |
| Apuntes externos | `PRG-APUNTES-OTROS` |
| Recursos | `PRG-RECURSOS` |
| Chuletas | `PRG-CHEATSHEETS` |
| Juegos | `PRG-JUEGOS` |
| Retos | `PRG-RETOS` |
| Cursos | `PRG-CURSOS` |
| Libros | `PRG-LIBROS` |
| Jueces online | `PRG-JUECES` |

### Comandos

```bash
# Previsualizar localmente
open index.html

# Comprobar que el JSON es válido
python -m json.tool recursos.json > /dev/null && echo "OK"

# Commit
git add index.html recursos.json
git commit -m "feat: añadir nuevos recursos"
git push
```

> Todos los JS externos se cargan desde CDN — no se commitearn.
> `.gitignore` bloquea `*.js` salvo `recursos.json`.
